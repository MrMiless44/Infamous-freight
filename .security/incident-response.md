# Incident Response Plan

## Security Incident Definition

Examples:

- data breach
- unauthorized access
- leaked credentials
- dependency vulnerability
- CI compromise

---

# Incident Response Process

1. Detect
2. Contain
3. Investigate
4. Remediate
5. Communicate
6. Post-mortem

---

# Immediate Containment

Possible actions:

- revoke tokens
- rotate secrets
- disable affected endpoints
- pause deployments
- block compromised accounts

---

# Investigation

Collect:

- logs
- request traces
- deployment history
- audit logs

---

# Recovery

- patch vulnerability
- redeploy services
- restore affected systems

---

# Post-Incident Review

Create a report documenting:

- root cause
- impact
- timeline
- lessons learned
- mitigation steps
