# Infamous Freight Security Policy

Infamous Freight is an AI-powered freight and logistics automation platform.
Security is critical because the platform processes shipment data, carrier records,
and multi-tenant business information.

This document explains how to report security vulnerabilities responsibly.

---

# Supported Versions

| Version | Supported |
|--------|-----------|
| main | ✅ |
| latest release | ✅ |
| older versions | ❌ |

Only the most recent release branch and `main` are supported with security fixes.

---

# Reporting a Vulnerability

If you discover a security vulnerability, please report it privately.

DO NOT open a public issue.

Send a report containing:

- description of the vulnerability
- steps to reproduce
- possible impact
- proof-of-concept if available
- suggested mitigation if known

Contact:

security@infamousfreight.com

or open a **private security advisory** through GitHub.

---

# Response Timeline

| Step | Target Time |
|-----|-------------|
| Initial response | 48 hours |
| Confirmation | 3–5 days |
| Fix preparation | varies |
| Public disclosure | after fix is released |

---

# Disclosure Policy

We follow **responsible coordinated disclosure**:

1. vulnerability reported privately
2. issue validated
3. fix prepared
4. patch released
5. disclosure published

Researchers will be credited unless anonymity is requested.

---

# Security Scope

This policy applies to:

- API services
- Web application
- Mobile app
- infrastructure code
- CI/CD pipelines
- authentication systems

---

# Out of Scope

The following are not considered security vulnerabilities:

- rate limits
- spam
- UI issues
- outdated browsers
