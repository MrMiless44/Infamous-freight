## 🏥 Incident Response Quick Reference

### If Error Rate Spikes > 5% (CRITICAL)
1. **Immediately**: Page on-call engineer
2. **Within 5 min**: Identify error source (logs, monitoring)
3. **Within 30 min**: Implement fix or execute rollback
4. **Within 24 hours**: Document root cause

### If Latency Spikes > 2s (HIGH)
1. Check database query performance
2. Check cache hit rate
3. Increase resources if needed
4. Review external service delays

### If Memory Usage > 90% (HIGH)
1. Check for memory leaks
2. Restart services if safe
3. Increase server RAM
4. Review logs for issues

### If Disk Usage > 90% (CRITICAL)
1. Check disk space
2. Clean up logs or backups
3. Add more disk space
4. Monitor growth rate
